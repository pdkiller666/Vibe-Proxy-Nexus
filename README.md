# Vibe Proxy Nexus

Приватный VPN-сервис по приглашениям: веб-панель для регистрации,
оплаты подписки и выдачи ключей доступа к VPN на протоколе VLESS
(Xray-core), поверх WebSocket+TLS на обычном веб-домене.

## Возможности

- **Регистрация по приглашению** — email + пароль, вход только по реферальной ссылке (`/sign-up?ref=КОД`) от существующего пользователя. Сессии в базе данных, восстановление пароля по ссылке.
- **Профиль** — смена имени, email и пароля (с подтверждением текущим паролём).
- **Тарифы и оплата** — выбор плана, поддерживаются два способа оплаты:
  - **ЮMoney (карта / SberPay)** — автоматическое подтверждение через QuickPay и HMAC-вебхук.
  - **СБП (ручной перевод)** — пользователь отправляет деньги, загружает скриншот перевода (обязательно), добавляет комментарий (необязательно), нажимает «Я оплатил(а)»; администратор подтверждает вручную.
  - Поддерживаются ежемесячные планы с фиксированной ценой и почасовые тарифы с автосписанием с баланса.
- **Активный тариф на странице планов** — текущая подписка выделена зелёным бейджем, кнопка заблокирована («Текущий тариф»), чтобы исключить случайное повторное подключение.
- **Настраиваемые СБП-реквизиты** — администратор может задать:
  - прямую ссылку на платёж в СБП-приложении банка;
  - QR-код для оплаты (загружается в панели, отдаётся клиентам как изображение);
  - переключатель видимости блока с номером/банком/получателем.
- **Баланс и пополнение** — внутренний кошелёк пользователя; пополняется через СБП-перевод (подтверждается админом); используется для почасовых тарифов и доп. устройств; история операций с балансом отображается на странице **Платежи**.
- **VPN-ключи и устройства** — выдача/отзыв VLESS-ключей (1 ключ = 1 устройство, лимит задаётся тарифом). Каждый ключ можно именовать и описать. Пользователи с достаточным балансом могут докупить дополнительные слоты устройств.
- **Самообновляющаяся ссылка-подписка** — одна стабильная ссылка агрегирует все активные ключи; VPN-клиенты (Happ, v2rayNG) подтягивают её автоматически — не нужно вручную обновлять конфиг при смене узлов.
- **Трафик** — лимит трафика на период подписки (если задан тарифом); превышение — автоматически отзываются ключи. В админке: детальная статистика UP/DOWN по ключу за жизнь и за период.
- **Автопродление и автопросрочка** — подписка честно продлевается от даты окончания текущей; просроченная — автоматически отзывает ключи (если нет другой активной подписки).
- **Реферальная программа** — комиссия в % от оплаченных подписок рефералов зачисляется в баланс реферера автоматически при подтверждении платежа.
- **Поддержка** — встроенная тикет-система: пользователь создаёт тикет, переписка доступна как пользователю, так и любому администратору.
- **Пробный период** — опционально, настраивается в админке (кол-во дней).
- **Статус активности (три состояния):** «на сайте» (запрос к API за последние 5 мин) / «использует VPN» (трафик за последние 10 мин) / «оффлайн». При одновременной активности побеждает более свежий сигнал.
- **Админ-панель** — очередь платежей (подтверждение/отклонение со скриншотом), CRUD тарифов и VPN-нод (с лимитом пользователей на ноду), управление ролями, настройки СБП (ссылка, QR-код, показ реквизитов, телефон/банк/получатель), настройки реферальной программы и почасового тарифа, сброс паролей пользователей, аналитика (онлайн сейчас, новые пользователи за 7/30 дней, распределение по тарифам, 14-дневный график дохода, доход за 30 дней).

## Стек

- **Монорепозиторий:** pnpm workspaces, Node.js 24, TypeScript 5.9
- **Backend:** Express 5, PostgreSQL + Drizzle ORM, Zod-валидация
- **Frontend:** React + Vite, wouter, TanStack Query
- **API:** контракт в OpenAPI, клиентские хуки и Zod-схемы генерируются через Orval
- **VPN:** Xray-core (VLESS), транспорт — WebSocket поверх обычного HTTPS-домена (без Reality и без сырого TCP)
- **Деплой:** один Docker-образ на Amvera Cloud (frontend + backend + Xray-core в одном контейнере, управляются через `supervisord`)
- **Платежи:** ЮMoney (автоматически) + СБП ручной перевод (подтверждается администратором)

## Разработка

```sh
pnpm install
pnpm --filter @workspace/api-server run dev   # backend, порт из $PORT
pnpm --filter @workspace/vpn-portal run dev   # frontend
```

Полезные команды:

```sh
pnpm run typecheck                                     # типы по всему монорепо
pnpm run build                                          # typecheck + сборка
pnpm --filter @workspace/api-spec run codegen           # перегенерировать API-хуки/схемы из openapi.yaml
pnpm --filter @workspace/db run push                    # применить изменения схемы БД (dev)
```

## Деплой

```sh
./deploy.sh "Сообщение на русском о том, что изменилось"
```

Пушит текущее рабочее дерево в GitHub (через REST API, так как обычный
`git push` из агентского шелла заблокирован), что запускает автосборку
и деплой на Amvera по корневому `Dockerfile`.

Продакшен-домены: `https://vibeproxynexus-pdkiller666.waw0.amvera.tech/` и `https://vpnexus.pro/`.
