# Vibe Proxy Nexus

Приватный VPN-сервис по приглашениям: веб-панель для регистрации,
оплаты подписки и выдачи ключей доступа к VPN на протоколе VLESS
(Xray-core), поверх WebSocket+TLS на обычном веб-домене.

## Возможности

- **Регистрация и вход** — email + пароль, сессии в базе данных,
  восстановление пароля по ссылке.
- **Профиль** — смена имени, email и пароля (с подтверждением текущим
  паролем).
- **Тарифы и оплата** — выбор плана, оплата через СБП (ручной перевод,
  подтверждается администратором).
- **VPN-ключи** — выдача/отзыв отдельных VLESS-ключей или одна
  самообновляющаяся ссылка-подписка для клиентских приложений
  (Happ, v2rayNG и т. п.).
- **Автопродление и автопросрочка** — подписка честно продлевается от
  даты окончания текущей, а просроченная — автоматически отзывает ключи.
- **Админ-панель** — очередь платежей, тарифы, VPN-ноды (с лимитом
  пользователей на ноду), роли пользователей, настройки СБП, аналитика
  (сколько пользователей онлайн, новые пользователи за 7/30 дней,
  распределение по тарифам, график дохода).

## Стек

- **Монорепозиторий:** pnpm workspaces, Node.js 24, TypeScript 5.9
- **Backend:** Express 5, PostgreSQL + Drizzle ORM, Zod-валидация
- **Frontend:** React + Vite, wouter, TanStack Query
- **API:** контракт в OpenAPI, клиентские хуки и Zod-схемы генерируются
  через Orval
- **VPN:** Xray-core (VLESS), транспорт — WebSocket поверх обычного
  HTTPS-домена (без Reality и без сырого TCP)
- **Деплой:** один Docker-образ на Amvera Cloud (frontend + backend +
  Xray-core в одном контейнере, управляются через `supervisord`)

## Разработка

```sh
pnpm install
pnpm --filter @workspace/api-server run dev   # backend, порт 5000
pnpm --filter @workspace/vpn-portal run dev   # frontend
```

Полезные команды:

```sh
pnpm run typecheck                                    # типы по всему монорепо
pnpm run build                                         # typecheck + сборка
pnpm --filter @workspace/api-spec run codegen           # перегенерировать API-хуки/схемы из openapi.yaml
pnpm --filter @workspace/db run push                    # применить изменения схемы БД (dev)
```

## Деплой

```sh
./deploy.sh "Сообщение на русском о том, что изменилось"
```

Пушит текущее рабочее дерево в GitHub (через REST API, так как обычный
`git push` из агентского шелла заблокирован), что запускает автосборку
и деплой на Amvera по корневому `Dockerfile`.
