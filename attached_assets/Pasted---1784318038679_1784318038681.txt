## 🛡️ Главная точка контроля: Свитч здоровья фонового биллинга в `healthz`

Так как проект теперь полностью usage-based (деньги за почасовые тарифы списываются микро-шагами каждые 5 минут на основе дельты gRPC-трафика), стабильность фоновых интервалов Node.js стала напрямую влиять на ваш кошелек.

Если в процессе долгой работы монолита (например, из-за минутного провисания внешней БД) один из встроенных циклов `setInterval` («тиков») выбросит необработанную ошибку, сам таймер в Node.js может тихо отключиться, хотя веб-сайт продолжит открываться. В итоге:

* `trafficPolling.ts` перестанет опрашивать Xray, и графики трафика у пользователей «замрут».
* `hourlyBilling.ts` перестанет списывать копейки с балансов, и пользователи начнут сидеть в VPN абсолютно бесплатно.

### Предлагаемое решение:

Поскольку у вас уже есть стандартный эндпоинт `GET /api/healthz`, его нужно превратить в «умный предохранитель»:

1. **Внедрить in-memory маркеры активности:**
В файлы `src/lib/hourlyBilling.ts` и `src/lib/trafficPolling.ts` добавить обновление глобальных переменных при каждом успешном прогоне джобы:
```typescript
// Внутри функции тика почасового биллинга
global.lastHourlyBillingTick = Date.now();

// Внутри функции тика опроса трафика Xray
global.lastTrafficPollingTick = Date.now();

```


2. **Модернизировать роут `src/routes/health.ts`:**
Переписать проверку так, чтобы она валидировала не просто общую доступность процесса, но и проверяла, живы ли наши «сердцебиения» (heartbeats):
```typescript
router.get('/healthz', async (req, res) => {
  const now = Date.now();
  const FIVE_MINUTES = 5 * 60 * 1000;
  const FIFTEEN_MINUTES = 15 * 60 * 1000;

  const isTrafficJobAlive = global.lastTrafficPollingTick && (now - global.lastTrafficPollingTick < FIVE_MINUTES);
  const isBillingJobAlive = global.lastHourlyBillingTick && (now - global.lastHourlyBillingTick < FIFTEEN_MINUTES);

  if (!isTrafficJobAlive || !isBillingJobAlive) {
    // Если хотя бы одна критическая фоновая задача зависла — отдаем 503
    return res.status(503).json({
      status: "error",
      reason: "Background jobs stale",
      trafficAlive: !!isTrafficJobAlive,
      billingAlive: !!isBillingJobAlive
    });
  }

  return res.status(200).json({ status: "healthy" });
});

```



### 🎯 В чем профит для вас?

Amvera Cloud непрерывно опрашивает эндпоинт `containerPort/api/healthz` (Health Check). Если бэкенд отдаст `503`, платформа поймет, что монолит находится в деградации, и автоматически мягко перезагрузит Docker-контейнер. Система сама починит себя за 20 секунд, а вы не потеряете ни рубля со списаний.
