### 📝 Обновленное ТЗ: Подготовка слоя мультиузловости (Локальный WS + Удаленный WS)

**Цель:** Реализовать поддержку удаленных VPN-серверов по схеме «Центральный монолит на Amvera + внешние VPS ноды», используя протокол **VLESS + WebSocket + TLS** как основной и самый стабильный.

#### 1. Расширение схемы БД (`lib/db/src/schema/`)

* Удалить старые неиспользуемые поля `panelUrl`, `panelLogin`, `panelPassword` из таблицы `vpn_nodes`.


* Добавить в таблицу `vpn_nodes` два новых поля:


* `managementApiUrl` (text, nullable) — URL удаленного Management API ноды. Если `null` — узел считается локальным.


* `managementApiSecret` (text, nullable) — секретный токен для авторизации (`X-Management-Secret`).




* *Важно:* Все изменения на проде должны накатываться через `heal-schema.mjs` перед основным `drizzle-kit push`.



#### 2. Развилка логики в `keyIssuance.ts`

* Внутри механизма выдачи ключей переписать проверку типа ноды. Вместо хардкода или проверки хостов ориентироваться строго на поле: `node.managementApiUrl != null`.


* Если `managementApiUrl` задан, бэкенд должен отправлять асинхронные HTTP-запросы к удаленной ноде:


* `POST {managementApiUrl}/clients` — для добавления нового UUID.


* `DELETE {managementApiUrl}/clients/{uuid}` — для отзыва ключа.


* Запросы должны быть авторизованы заголовком `X-Management-Secret: {managementApiSecret}`.




* Если поле пустое — выполнять текущую стабильную логику правки локального конфига на диске Amvera.



#### 3. Доработка Python API удаленного узла (`deploy/amvera-vpn-node/`)

* В файл `api_server.py` необходимо добавить недостающий эндпоинт `GET /stats`.


* Метод должен обращаться к локальному gRPC Stats API Xray на этой ноде, собирать текущие счетчики трафика и отдавать их центральному бэкенду в формате JSON `Map<uuid, {up, down}>`.


* Эндпоинт должен быть жестко защищен проверкой токена `X-Management-Secret`.



#### 4. Обновление `trafficPolling.ts` для сбора метрик с внешних нод

* Модернизировать минутный цикл опроса трафика.


* Скрипт должен выбирать из БД все активные ноды. Для локальной ноды опрос gRPC идет напрямую (как сейчас), а для внешних нод (`managementApiUrl != null`) бэкенд делает HTTP-запрос к `{managementApiUrl}/stats`.


* Полученные массивы дельт трафика со всех серверов должны прозрачно передаваться в готовую функцию `applyTrafficDeltas()`, чтобы почасовой биллинг и лимиты работали без сбоев независимо от локации сервера.



#### 5. Сохранение формата ссылок в `vless.ts`

* Функция `buildVlessLink()` должна продолжать генерировать ссылки под **VLESS + WebSocket + TLS** для новых нод, так как этот протокол показал 100% совместимость со всеми заблокированными сервисами (включая Gemini), в отличие от капризного Reality. Хост и SNI в ссылке должны динамически браться из полей созданной ноды.
